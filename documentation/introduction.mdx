## Overview

The GOES Health API is a REST-first service that delivers outdoor risk intelligence as easy-to-consume JSON. Each endpoint is:

- Location-aware: latitude/longitude in every request
- Real-time: continuously updated location risks for time-sensitive decisions
- Developer-friendly: predictable query parameters and standard HTTP status codes
- Mobile-ready: lightweight responses designed for low-latency cell networks

Whether you are displaying a compact widget or powering a full-featured trip-planning assistant, the API lets you select exactly the data you need and nothing more.

## API Endpoints

Our API provides three core endpoints to access outdoor risk intelligence:

<CardGroup cols={1}>
  <Card title="Environmental Risks">
    ```bash
    GET /v1/risks
    ```
    Retrieve real-time environmental health risks for any latitude/longitude. Pass `location=LAT,LON` & `api_key=KEY` (optional `ids` to filter). The response returns an array of risks with `id`, `title`, and actionable safety tips.
  </Card>

  <Card title="Wildlife Risks">
    ```bash
    GET /v1/wildlife
    ```
    Retrieve wildlife-related risks for the specified coordinate. Required `api_key` and `location`; optional `ids` to filter. Response contains a `responseId` plus an array of wildlife risks where each item includes `id`, `title`, prevalence `score` (0–3), and safety tips.
  </Card>

  <Card title="Overview & Summary">
    ```bash
    GET /v1/overview
    ```
    Generate an AI-curated overview that combines environmental risks, wildlife risks, and active NOAA alerts. Personalize using `length`, `activity`, `age`, `outdoor_experience`, `context`, and `exclude` query parameters. The response returns a `responseId`, human-readable summary, detailed risks (with `severity` 0–4), and any active alerts.
  </Card>
</CardGroup>

## Authentication

All API requests require an API key as a query parameter:

```bash
curl "https://api.goes.health/v1/risks?location=37.7325,-119.5581&api_key=YOUR_API_KEY"
```

[Request an API key here](mailto:api@goes.health?subject=API%20Key%20Request)

## Rate Limits

- 600 requests per minute by default
- Contact us for higher volume needs
